## 已知限制（工程範圍決策，供審核時一併參考）

- **不修改 `useSlashDetector` hook**：曲球/蝴蝶球的彎曲與飄忽只做「有界視覺偏移」疊加在真實判定路徑上（偏移量 < 判定半徑），因此視覺上球會彎，但實際判定仍是 hook 原生的等速拋物線。偏移刻意壓在容錯範圍內，避免「看到哪裡揮卻打不到」的不公平感，但也代表曲球的彎曲程度並非物理精確模擬。
- **「揮速」未落地**：規格提及的「揮棒時機偏差（avg_reaction_ms）」已實作，但「揮速」（揮臂速度）因 `useSlashDetector` 未對外提供即時腕點速度（僅約 10Hz 取樣的軌跡陣列），且不允許新增 Supabase 欄位，若勉強估算精準度低、又要呈現給治療師/家屬看，決定**不做**，留待未來若要做需先評估是否該讓 hook 對外多回傳一個瞬時速度欄位。
- **各球種命中數只在結算頁顯示、未落庫**：`SaveSessionPayload`／`/api/game/save` 沒有對應欄位，依契約不新增 schema，故此指標僅供玩家/單場即時參考，不進治療師後台歷史分析。
- **患側（左打/右打）只影響打擊區水平位置與曲球彎曲方向，不落庫**：同樣受限於既有 payload schema 沒有對應欄位；影響會間接反映在 zone_heatmap／left_hits／right_hits 的分布上。
