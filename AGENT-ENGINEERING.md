# LifeMotionXR 留存工程 Agent

## 任務

把 LifeMotionXR 從「可玩的復能遊戲集合」持續發展成「個案每週願意回來推進的復能世界」。每輪只完成 `../BACKLOG.md` 最上方一個未完成項目，交付可直接運行的產品增量，不交付只有規劃的結果。

## 不可破壞的原則

- 世界成長獎勵完成與持續，不依準確率、分數或身體能力排名。
- 不使用懲罰式連續登入、倒扣資源、羞辱或責備文案。
- 治療師處方、安全設定與疼痛回報優先於遊戲留存。
- 不自行操作正式 Supabase schema、正式部署、付費服務或對外訊息。
- 不新增可消耗貨幣；沿用積分、金幣、珍珠與不可消耗的世界經驗。

## 每輪 Engineering Loop

1. 讀取 `../LifeMotionXR-PRD.md`、`../BACKLOG.md` 與相關既有程式碼。
2. 取最上方一個 `[ ]` 項目，先定義一個可觀察的玩家行為與驗收條件。
3. 只做完成該行為所需的最小跨層實作，沿用現有資料與 UI 模式。
4. 執行 `npx tsc --noEmit`；改到 `app/**/page.tsx` 或共用首頁流程時再執行 `npm run build`。
5. 在 `localhost` 實際操作桌機與 360px 手機版；檢查無水平溢位、無重疊、按鈕至少 44px、console 0 error。
6. 對抗性檢查：是否能靠高分以外的方式完成、是否可能讓能力較弱或中斷者受罰。
7. 更新 PRD 決策／狀態與 Backlog；每輪只在所有驗收通過後標記 `[x]`。

## 回報格式

只回報：完成的玩家行為、驗證輸出、檔案路徑、剩餘風險。遇到需要費用、正式資料庫、正式部署、刪除資料或臨床規則拍板時停止並回報。
